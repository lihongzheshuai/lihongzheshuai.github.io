## luogu-P1048 [NOIP 2005 普及组] 采药

### 题目要求

#### 题目描述

> 辰辰是个天资聪颖的孩子，他的梦想是成为世界上最伟大的医师。为此，他想拜附近最有威望的医师为师。医师为了判断他的资质，给他出了一个难题。医师把他带到一个到处都是草药的山洞里对他说："孩子，这个山洞里有一些不同的草药，采每一株都需要一些时间，每一株也有它自身的价值。我会给你一段时间，在这段时间里，你可以采到一些草药。如果你是一个聪明的孩子，你应该可以让采到的草药的总价值最大。"
>
> 如果你是辰辰，你能完成这个任务吗？

#### 输入格式

> 第一行有 $2$ 个整数 $T$（$1 \le T \le 1000$）和 $M$（$1 \le M \le 100$），用一个空格隔开，$T$ 代表总共能够用来采药的时间，$M$ 代表山洞里的草药的数目。
>
> 接下来的 $M$ 行每行包括两个在 $1$ 到 $100$ 之间（包括 $1$ 和 $100$）的整数，分别表示采摘某株草药的时间和这株草药的价值。

#### 输出格式

> 输出在规定的时间内可以采到的草药的最大总价值。

#### 输入输出样例 #1

##### 输入 #1

```plaintext
70 3
71 100
69 1
1 2
```

##### 输出 #1

```plaintext
3
```

#### 说明/提示

##### 数据范围

- 对于 $30\%$ 的数据，$M \le 10$
- 对于全部的数据，$M \le 100$

##### 题目来源

### 题目分析

本题是最经典的 **01 背包问题**。将采药的场景翻译为背包模型：

- **背包容量** $\rightarrow$ 总时间 $T$
- **物品** $\rightarrow$ 每株草药
- **物品重量** $\rightarrow$ 采摘该株草药所需的时间
- **物品价值** $\rightarrow$ 该株草药的价值
- **每种物品最多选 1 件** $\rightarrow$ 每株草药只有一株，采了就没了

#### 1. 什么是 01 背包？

01 背包是背包问题中最基本的类型：

> 给定 $M$ 件物品和一个容量为 $T$ 的背包。第 $i$ 件物品的重量为 $w_i$，价值为 $v_i$。每件物品**最多选一次**（要么选，要么不选，即 0 或 1），求在不超过背包容量的前提下，能获得的最大总价值。

#### 2. 状态定义

定义 $dp[j]$ 表示：**背包容量为 $j$ 时，能获得的最大总价值**。

初始状态：$dp$ 数组全部初始化为 $0$，表示什么都不装时价值为 $0$。

#### 3. 状态转移方程

对于第 $i$ 件物品（耗时 $w_i$，价值 $v_i$），对每个容量 $j$（从 $T$ 到 $w_i$）：

$$dp[j] = \max(dp[j],\ dp[j - w_i] + v_i)$$

**方程含义**：面对第 $i$ 株草药和容量 $j$，有两种选择：

1. **不采**：价值保持 $dp[j]$ 不变
2. **采**：花费时间 $w_i$，获得价值 $v_i$，剩余容量 $j - w_i$ 下的最大价值为 $dp[j - w_i]$

取两者中较大的那个。

#### 4. 样例验证

样例：$T = 70, M = 3$

| 草药编号 | 采摘时间 $w$ | 价值 $v$ |
| :------: | :----------: | :------: |
|   $1$    |     $71$     |  $100$   |
|   $2$    |     $69$     |   $1$    |
|   $3$    |     $1$      |   $2$    |

逐个处理每件物品：

- **草药 $1$**（$w=71, v=100$）：由于 $w = 71 > T = 70$，容量不足，无法选择，跳过
- **草药 $2$**（$w=69, v=1$）：只有 $j \ge 69$ 时才能选，$dp[69] = 1, dp[70] = 1$
- **草药 $3$**（$w=1, v=2$）：从大到小更新
  - $dp[70] = \max(dp[70], dp[69] + 2) = \max(1, 1 + 2) = 3$
  - $dp[69] = \max(dp[69], dp[68] + 2) = \max(1, 0 + 2) = 2$
  - ……
  - $dp[1] = \max(dp[1], dp[0] + 2) = \max(0, 0 + 2) = 2$

最终 $dp[70] = 3$，对应选择草药 $2$（价值 $1$）和草药 $3$（价值 $2$），总时间 $69 + 1 = 70 \le 70$，与样例输出一致。

#### 5. 完整算法流程

1. 读入总时间 $T$ 和草药数目 $M$
2. 初始化 $dp$ 数组为全 $0$
3. 对每株草药 $i$（外层循环），从大到小遍历容量 $j$（从 $T$ 到 $w_i$），执行状态转移
4. 最终答案为 $dp[T]$

#### 6. 复杂度分析

- **时间复杂度**：$O(M \times T)$，双层循环。$M \le 100, T \le 1000$，运算次数最多 $10^5$，非常快
- **空间复杂度**：$O(T)$，一维滚动数组

#### 7. 注意事项

- 当某株草药的采摘时间 $w_i > T$ 时，该草药永远无法被选取，循环自然跳过（内层循环条件 $j \ge w_i$ 不满足）
- 本题数据范围很小（$M \le 100, T \le 1000$），01 背包的标准做法绰绰有余
- 最大总价值不超过 $M \times \max(v_i) = 100 \times 100 = 10000$，`int` 类型完全足够

### 示例代码

#### 方法一：二维 DP（最直观的写法）

用二维数组 $dp[i][j]$ 表示"考虑前 $i$ 件物品，背包容量为 $j$ 时的最大价值"。每个格子只看上一行的数据来计算，逻辑清晰，容易理解。

```cpp
#include <iostream>
#include <algorithm>

// dp[i][j] 表示考虑前 i 件物品，背包容量为 j 时的最大价值
int dp[105][1005];

// 存储每件物品的耗时和价值
int w[105], v[105];

int main() {
    int T, M;
    std::cin >> T >> M;

    // 读入所有物品的信息
    for (int i = 1; i <= M; i++) {
        std::cin >> w[i] >> v[i];
    }

    // dp[0][j] 全为 0（没有物品可选时，价值为 0），数组默认初始化即可

    // 外层循环：依次考虑第 1 ~ M 件物品
    for (int i = 1; i <= M; i++) {
        // 内层循环：枚举背包容量 j 从 0 到 T
        for (int j = 0; j <= T; j++) {
            // 默认不选第 i 件物品，沿用上一行的值
            dp[i][j] = dp[i - 1][j];

            // 如果容量够放下第 i 件物品，尝试选它，取更大的值
            if (j >= w[i]) {
                dp[i][j] = std::max(dp[i][j], dp[i - 1][j - w[i]] + v[i]);
            }
        }
    }

    // dp[M][T] 即为考虑所有 M 件物品、背包容量为 T 时的最大价值
    std::cout << dp[M][T] << std::endl;
    return 0;
}
```

二维写法的好处是逻辑非常清晰：`dp[i-1][j]` 和 `dp[i-1][j-w[i]]` 明确地告诉你"我在看上一行的数据"，不会有任何歧义。缺点是空间占用较大（$O(M \times T)$），但本题数据范围很小，完全没有问题。

#### 方法二：一维 DP（空间优化版）

观察方法一的代码可以发现：计算第 $i$ 行时，我们只用到了第 $i-1$ 行，更早的行都用不到了。所以可以把二维数组压缩成一维数组，只保留一行，每处理一件物品就在这一行上直接覆盖更新。

**关键点**：内层循环必须**从大到小**遍历容量，原因在下一节详细解释。

```cpp
#include <iostream>

// dp[j] 表示在总时间为 j 的情况下，能采到的草药的最大总价值
int dp[1005];

int main() {
    int T, M;
    std::cin >> T >> M;

    // 外层循环：依次考虑每一株草药
    for (int i = 0; i < M; i++) {
        int w, v;
        std::cin >> w >> v;

        // 内层循环：从大到小遍历容量（01 背包的关键）
        // 保证每株草药最多只被选一次
        for (int j = T; j >= w; j--) {
            // 状态转移：采或不采第 i 株草药
            if (dp[j - w] + v > dp[j]) {
                dp[j] = dp[j - w] + v;
            }
        }
    }

    // dp[T] 即为在总时间 T 内能采到的草药的最大总价值
    std::cout << dp[T] << std::endl;
    return 0;
}
```

---

### 为什么内层循环要从大到小遍历？

看完上面的代码，你可能会有一个疑问：内层循环写的是 `for (int j = T; j >= w; j--)`，为什么 $j$ 要从 $T$ **递减**到 $w$，而不是从 $w$ **递增**到 $T$？

这个问题的本质非常简单。我们可以用 **“旧数据”** 和 **“新数据”** 的概念，从纯逻辑角度来理解。

在将二维 DP 压缩为一维数组后，我们只有一个 `dp` 数组。当我们在考虑 **“要不要采摘当前这株草药（为了直观，我们假设它是个苹果）”** 时：

- **01 背包的规则**：苹果只有**一个**，要么不拿，要么拿一个。
- **核心逻辑**：如果决定拿，我们必须用**“还没放苹果之前的旧状态”**来加上苹果的价值。

---

#### 1. 如果“从小到大”遍历（正序）

我们会先更新容量小的，再更新容量大的。

1. **算 `dp[2]`**：放了一个苹果。此时 `dp[2]` 变成了**已经含有苹果的“新数据”**。
2. **算 `dp[4]`**：根据公式，`dp[4]` 要参考 `dp[2]`。
   - `dp[4] = dp[2]`（新数据） $+$ 苹果的价值
   - 因为 `dp[2]` 里已经放过一个苹果了，你现在又加一个，结果就是：**放了两个苹果！**

**结论**：正序遍历会导致“同一个物品被重复放入”，这违背了 01 背包“每个物品最多选一次”的原则（这其实是**完全背包**的逻辑）。

---

#### 2. 如果“从大到小”遍历（逆序）

我们会先更新容量大的，再更新容量小的。

1. **算 `dp[4]`**：根据公式，`dp[4]` 要参考 `dp[2]`。
   - 因为我们是从右往左更新，此时 `dp[2]` **还没轮到被更新**。
   - 所以，这时的 `dp[2]` 依然是上一轮的**“不含苹果的旧数据”**。
   - `dp[4] = dp[2]`（旧数据） $+$ 苹果的价值。这样，我们就只放了**一个**苹果，非常完美！
2. **算 `dp[2]`**：最后我们才更新 `dp[2]`，把它变成含有苹果的“新数据”。但此时大容量的 `dp[4]` 早就计算完了，不会受到任何影响。

**结论**：逆序遍历保证了你在计算大容量 `dp[j]` 时，用到的 `dp[j - w]` 都是**还没考虑当前物品的纯净旧值**，从而确保每株草药**最多只被选择一次**。

### 拓展思考

#### 01 背包与完全背包的代码对比

将本题（01 背包）和 B2174（完全背包）的核心代码放在一起，差异一目了然：

```diff
  for (int i = 0; i < n; i++) {
      int w, v;
      std::cin >> w >> v;
-     // 完全背包：正序遍历，允许重复选取
-     for (int j = w; j <= V; j++) {
+     // 01 背包：逆序遍历，每件物品最多选一次
+     for (int j = T; j >= w; j--) {
          if (dp[j - w] + v > dp[j]) {
              dp[j] = dp[j - w] + v;
          }
      }
  }
```
